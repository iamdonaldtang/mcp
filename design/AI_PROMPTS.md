# Claude AI 设计助手提示词库

## 快速审查 (最常用)
用途：完成一个屏幕后，快速获得反馈
我是在用Pencil设计TaskOn Onboarding。
请审查这个设计截图：
【视觉层级】最重要的元素最显眼吗？用户的视线流是否正确？
【可读性】文字大小、对比度、行距是否良好？
【导向性】用户是否清楚下一步？CTA足够显眼吗？
【品牌一致】是否符合TaskOn风格？
【一句改进】如果只能改一处，应该改什么？
背景：我遵循简洁>复杂, 故事>功能列表的设计原则。

## 角色适配检查
用途：确保设计符合特定角色的需求
这个屏幕是为 [CMO / Community Manager / CTO / CEO] 设计的。
请检查：

这个角色的痛点有没有被直接触碰？
"快赢"足够清晰吗？

CMO的快赢：30分钟启动Quest
Community的快赢：一键配置Points系统
CTO的快赢：无代码嵌入组件
CEO的快赢：ROI计算器


文案措辞是否符合这个角色？
CTA是否真的会吸引这个角色？


## 文案优化
用途：改进按钮文本、标题等
这是我的CTA文案：[粘贴文案]
请优化成：

更有吸引力（action-oriented，不是陈述）
更简洁（<10个字）
更符合Web3/加密行业的语言
有紧迫感（但不过度）

给出3个替代方案，并解释为什么这个更好。

## 用户流程检查
用途：确保流程够短、逻辑清晰
我设计的 [CMO Path / Community Path / ...] 前 3 个屏幕流程：
[上传3个截图，描述流程：屏幕1 → 屏幕2 → 屏幕3]
请检查：

这个流程是最短的吗？有没有可以删除的屏幕？
每个屏幕之间的逻辑转换是否自然？
会不会让用户感到困惑？
目标用户能在 <3分钟 内完成吗？

Onboarding应该快速，不应该教学。

背景：[描述或HEX值]
主色：[描述或HEX值]
文字：[描述或HEX值]

[上传截图]
请检查：

颜色对比度是否满足WCAG AA标准？（≥4.5:1）

是否与TaskOn品牌一致？
